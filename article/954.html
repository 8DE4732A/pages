Google Stadia 和 Immersive Stream for Games 所配的图形服务器即将升级支持 NVIDIA GPU。自 2019 年推出以来，Google Stadia 及其底层的沉浸式游戏流服务一直由 AMD 定制的 GPU 提供支持。研究人员普遍认为 Stadia 的服务器运行在 AMD 的 Radeon Pro V340 硬件或与之非常接近的硬件上。

根据爆料信息，Google 似乎计划让 Stadia 和 Immersive Stream 支持额外的图形硬件。这些信息来自 Stadia 的 Linux 内核修改版本。在上个月的代码更改中，Google已经为他们的自动化构建器（“Kokoro”）添加了一种方法来包含 NVIDIA GPU 的必要驱动程序。

在代码中写道：“添加对 kokoro 作业脚本的支持，以生成包含 UMD/KMD NVIDIA 模块和使用 NVIDIA gpu 的实例所需的相应支持文件的磁盘”。

目前，尚无任何线索表明 Stadia 和 Immersive Stream 可能会使用哪些特定的 NVIDIA 显卡。虽然 AMD 长期以来一直提供用于 Linux 设备的开源 GPU 驱动程序，但 NVIDIA 直到最近才开始提供相同的驱动程序。因此，新包含的用于 Stadia 的 NVIDIA 驱动程序（尚未）可供检查。尽管缺乏细节，但这里的含义相当有趣。
