面向所有频道，Windows Subsystem for Android（WSA）今天获得了 2205.40000.14.0 版本更新。本次更新带来了大量新功能，包括对 AV1 编解码器的支持、设置应用程序也获得了升级。以下是 WSA 7 月完整更新日志：

Advanced Networking

微软对 WSA 网络部分进行了重大调整。微软引入了 Advanced Networking（高级网络），它允许应用程序访问 ARM 计算机的本地网络设备，从而允许应用程序利用此功能。微软还启用了 IPv6 和 VPN 连接，增强了子系统的网络功能。

设置应用更新

微软更新了 Windows Subsystem for Android（WSA）的设置应用，整合了 Advanced Networking 选项。这意味着 IP 地址已从“设置”应用程序的“开发人员”部分中删除，因为子系统不再具有与您的计算机不同的 IP。

完整的更新日志

● 启用高级网络功能，包括应用程序访问 ARM 的本地网络设备

● 从“设置”应用中删除了 VM IP 地址。使用高级网络，现在 VM 的 IP 地址与主机/计算机 IP 地址相同。

● 修复了在最大化或调整大小时不可调整大小的应用内容

● 修复了在应用程序中使用鼠标和触控板滚动的问题

● Android May 内核补丁

● 标记为安全的 Android 窗口无法再截屏

● 改进网络浏览器启动

● 在充电时启用打盹和应用待机以提高节电效果

● ADB 调试提示重定向到 Windows 以提高安全性

● 更新到 Chromium WebView 101

● 图形修复，包括应用程序闪烁和图形损坏

● 视频播放修复

● AV1 编解码器支持

● 启用 IPv6 和 VPN 连接

● 提高了连接到容器中的虚拟 WIFI 的性能和可靠性

● 视频播放应用程序现在可以防止屏幕在 Windows 中关闭

遗憾的是，本次更新目前仅限于美国地区。以下是此新 WSA 版本 2205.40000.14.0 更新中的已知问题：

某些 VPN 可能不适用于高级网络。如果您使用 VPN 并发现 Android 应用程序没有网络连接，请在 Windows Subsystem for Android Settings 应用程序中禁用高级网络。
